    FileExports
    
  Function
  v1.0
  
    (FileExports __src)
    

    
  
    FileExports trims the AST for a Go source file in place such that
    
only exported nodes remain: all top-level identifiers which are not exported
    
and their associated information (such as type, initial value, or function
    
body) are removed. Non-exported fields and methods of exported types are
    
stripped. The File.Comments list is not changed.
    

    
FileExports reports whether there are exported declarations.
    

    
Go input arguments: (src *File)
    

    
Go return type: bool
    

    
Joker input arguments: [^(atom-of go.std.go.ast/File) src]
    

    
Joker return type: Boolean

    FilterDecl
    
  Function
  v1.0
  
    (FilterDecl __decl __f)
    

    
  
    FilterDecl trims the AST for a Go declaration in place by removing
    
all names (including struct field and interface method names, but
    
not from parameter lists) that don't pass through the filter f.
    

    
FilterDecl reports whether there are any declared names left after
    
filtering.
    

    
Go input arguments: (decl Decl, f Filter)
    

    
Go return type: bool
    

    
Joker input arguments: [^go.std.go.ast/Decl decl, ^go.std.go.ast/Filter f]
    

    
Joker return type: Boolean

    FilterFile
    
  Function
  v1.0
  
    (FilterFile __src __f)
    

    
  
    FilterFile trims the AST for a Go file in place by removing all
    
names from top-level declarations (including struct field and
    
interface method names, but not from parameter lists) that don't
    
pass through the filter f. If the declaration is empty afterwards,
    
the declaration is removed from the AST. Import declarations are
    
always removed. The File.Comments list is not changed.
    

    
FilterFile reports whether there are any top-level declarations
    
left after filtering.
    

    
Go input arguments: (src *File, f Filter)
    

    
Go return type: bool
    

    
Joker input arguments: [^(atom-of go.std.go.ast/File) src, ^go.std.go.ast/Filter f]
    

    
Joker return type: Boolean

    FilterPackage
    
  Function
  v1.0
  
    (FilterPackage __pkg __f)
    

    
  
    FilterPackage trims the AST for a Go package in place by removing
    
all names from top-level declarations (including struct field and
    
interface method names, but not from parameter lists) that don't
    
pass through the filter f. If the declaration is empty afterwards,
    
the declaration is removed from the AST. The pkg.Files list is not
    
changed, so that file names and top-level package comments don't get
    
lost.
    

    
FilterPackage reports whether there are any top-level declarations
    
left after filtering.
    

    
Go input arguments: (pkg *Package, f Filter)
    

    
Go return type: bool
    

    
Joker input arguments: [^(atom-of go.std.go.ast/Package) pkg, ^go.std.go.ast/Filter f]
    

    
Joker return type: Boolean
